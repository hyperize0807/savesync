## 🐛 자동 업데이트 재시작 문제 수정 (중요)

v1.0.3 / v1.0.4 에서 **'업데이트 확인'** 후, *PID를 찾는 터미널 창이 뜬 채로 멈추던* 문제를 수정했습니다. (교체 자체는 되어 있어 수동 재실행하면 업데이트되던 증상)

**원인** — 재시작 헬퍼(`.cmd`)의 결함 4가지가 겹쳐 있었습니다.

1. **`timeout` 무력화** — 헬퍼가 `stdin=DEVNULL`로 실행되는데, 이 경우 `timeout /t` 은 *"Input redirection is not supported"* 로 즉시 종료되어 모든 대기 루프가 사라짐
2. **PID 단일 대기의 한계** — `--onefile` exe는 부트로더(부모)+자식 프로세스가 **둘 다** exe를 잠그므로, 자식 PID만 기다리면 파일이 잠긴 채 교체가 실패
3. **보이는 터미널 창** — `DETACHED_PROCESS`로 띄워 콘솔 하위명령이 별도 터미널 창을 생성
4. **이중 CR** — `.cmd`가 텍스트 모드로 기록되며 줄 끝이 `\r\r\n`이 되어 배치 라벨/`goto`/변수를 오염

**수정**
- 모든 대기를 **`ping` 기반**으로 변경 (stdin과 무관하게 동작)
- 파일 잠금이 풀릴 때까지 **`move` 재시도로 교체** (재시도 20→60), PID 의존 제거
- `DETACHED_PROCESS` → **`CREATE_NO_WINDOW`** (보이던 멈춘 터미널 창 제거)
- `.cmd`를 `newline=''`로 기록해 **줄 끝 정상화**
- 적용 안내 팝업을 헬퍼 실행보다 **먼저** 표시
- 회귀 테스트 추가(timeout 미사용 / ping 사용 / ASCII 본문 보장) + 실제 스크립트 E2E 검증

## ⚠️ 기존 버전(v1.0.3 / v1.0.4) 사용자 필독

이 수정은 **실행 중인 프로그램 안의 업데이터 코드**가 고쳐져야 효력이 생깁니다.
따라서 **깨진 버전에서 v1.0.5로 올라오는 "첫 업데이트" 한 번은 여전히 옛 스크립트**를 사용하므로 동일하게 멈출 수 있습니다. 아래 중 한 가지로 v1.0.5에 도달하시면, **그 다음 업데이트부터는 자동으로 깔끔하게 동작**합니다.

**방법 A — 직접 교체 (권장, 가장 확실)**
1. 이 릴리스 하단의 **`SaveSync.exe`** 를 내려받기
2. 트레이에서 SaveSync **종료**
3. 기존 `SaveSync.exe` 를 내려받은 파일로 **덮어쓰기**
4. 실행 → 끝

**방법 B — 앱 내 업데이트 후 수동 재실행**
1. '업데이트 확인' → 업데이트 진행
2. PID를 찾는 터미널 창에서 멈추면, **그 창을 닫고**(또는 작업 관리자에서 종료)
3. `SaveSync.exe` 를 **수동으로 다시 실행** — 교체는 이미 끝나 v1.0.5로 실행됩니다

> 설정·인증 토큰·백업은 `%APPDATA%\SaveSync` 에 보관되므로, exe 를 교체해도 그대로 유지됩니다.
