## 🐛 자동 업데이트 후 재시작 실패 수정 (중요)

'업데이트 확인'으로 업데이트한 뒤, 새 버전이 시작되며 아래 오류로 멈추던 문제를 수정했습니다.

```
Failed to load Python DLL '...\_MEI######\python313.dll'.
LoadLibrary: 지정된 모듈을 찾을 수 없습니다.
```

**원인** — PyInstaller onefile 부트로더는 추출 폴더를 `_MEIPASS2` 같은 내부 환경변수로 전달합니다. 재시작 헬퍼(`.cmd`)를 띄울 때 실행 중 앱의 환경을 그대로 물려주는 바람에, 이 변수가 새로 실행되는 exe까지 상속되었습니다. 그 결과 새 부트로더가 재추출을 건너뛰고 **이미 삭제된 옛 `_MEI` 폴더**에서 DLL을 찾으려다 실패했습니다.

**수정** — 헬퍼를 실행할 때 `_MEI*`/`_PYI*` 로 시작하는 PyInstaller 내부 환경변수를 제거한 깨끗한 환경을 넘기도록 했습니다. 이제 새 exe가 자기 번들을 정상적으로 추출해 실행됩니다. (회귀 테스트 추가)

---

## ⚠️ 기존 버전(v1.0.6 이하) 사용자 필독

이 수정은 **실행 중인 프로그램 안의 업데이터 코드**가 고쳐져야 효력이 생깁니다. 따라서 **v1.0.7로 올라오는 "첫 업데이트" 한 번은 여전히 옛 코드**로 실행되어 위 오류가 날 수 있습니다. 이때 **교체 자체는 이미 끝나 있으므로** 아래 중 하나로 v1.0.7에 도달하면, **그 다음 업데이트부터는 자동으로 깔끔하게 동작**합니다.

**방법 A — 오류가 났다면**
1. 오류 팝업의 **확인**을 눌러 닫기
2. `SaveSync.exe` 를 **수동으로 다시 실행** — 교체는 끝나 있어 v1.0.7로 실행됩니다

**방법 B — 직접 교체 (가장 확실)**
1. 이 릴리스 하단의 **`SaveSync.exe`** 를 내려받기
2. 트레이에서 SaveSync **종료** 후, 기존 파일을 내려받은 파일로 **덮어쓰기** → 실행

> 설정·인증 토큰·백업은 `%APPDATA%\SaveSync` 에 보관되므로 exe 를 교체해도 그대로 유지됩니다.
