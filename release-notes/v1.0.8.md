## 🐛 인증 만료 시 '재인증' 자가 복구 수정

저장된 로그인 토큰이 만료·취소된 상태에서 **'Google Drive 연결 / 재인증'을 눌러도 계속 인증 실패**가 나던 문제를 수정했습니다.

**증상** — 동기화가 `invalid_grant: Token has been expired or revoked` 로 실패하고, 재인증 버튼을 눌러도 같은 실패가 반복됨. (이전에는 `%APPDATA%\SaveSync\token.json` 을 직접 지워야 복구됐습니다.)

**원인** — 인증 로직이, 토큰 파일이 남아 있으면 **항상 기존(폐기된) 토큰 갱신을 먼저 시도**하고 거기서 멈춰, 재인증을 눌러도 새 브라우저 인증 단계로 넘어가지 못했습니다.

**수정** — 토큰 갱신이 실패(폐기/만료)하면 **죽은 토큰을 버리고 자동으로 새 브라우저 인증으로 폴백**하도록 변경했습니다. 이제 토큰 파일을 직접 지우지 않아도 **'재인증' 한 번으로 복구**됩니다. (백그라운드 자동 동기화에서는 브라우저를 열지 않고 안전하게 건너뜁니다. 회귀 테스트 추가.)

> 설정·인증 토큰·백업은 `%APPDATA%\SaveSync` 에 보관되며, 업데이트로 exe 를 교체해도 그대로 유지됩니다.
