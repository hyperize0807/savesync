## 🐛 세이브 파일 패턴 필터 수정 (중요)

확장자와 포함 패턴을 **둘 다** 입력했을 때, 확장자만 맞으면 패턴이 달라도 동기화되던 문제를 수정했습니다.

**증상** — 예: 확장자 `.sav` + 포함 패턴 `G3P_II*` 로 설정해도 `G30001.sav`(확장자만 일치)가 업로드됨.

**원인** — 포함 규칙이 **확장자 OR 패턴**(둘 중 하나만 맞으면 포함)으로 동작.

**수정** — 입력한 포함 조건을 **모두 만족(AND)** 하도록 변경했습니다.
- 확장자 목록 안에서는 하나만 맞으면 됨(OR), 패턴 목록 안에서도 OR
- 단, **확장자와 패턴을 둘 다 입력하면 양쪽 모두** 만족해야 포함
- 한쪽만 입력하면 그 조건만 적용 / 둘 다 비우면 전체 포함
- 제외 패턴(`exclude_globs`)은 변함없이 최우선

| 파일 | 확장자 `.sav` | 패턴 `G3P_II*` | 결과 |
|---|---|---|---|
| `G3P_II_01.sav` | ✓ | ✓ | 포함 |
| `G30001.sav` | ✓ | ✗ | 제외 (이전엔 업로드되던 버그) |
| `G3P_II_save.dat` | ✗ | ✓ | 제외 |

> 회귀 테스트를 추가했습니다(재현 케이스·한쪽만 지정·다중 확장자·제외 우선).

### ⚠️ 동작 변화 안내
이전에 "`.sav` 전부 **또는** `save*` 전부"처럼 **합집합(OR)** 으로 쓰셨다면, 이제 교집합(AND)으로 좁혀집니다. 확장자와 패턴 중 **한쪽만** 입력해 쓰던 설정은 그대로 동작합니다.
